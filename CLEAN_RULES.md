# 清洗规则说明

## 设计理念：极简主义

工具的核心原则：**简单、稳定、不会出错**

- 上游增就增，上游删就删，上游变就变
- 不做复杂的过滤、测速、复活逻辑
- 信任上游，相信上游的变化

## 工作方式

1. 抓取上游原始 JSON，保存为留底文件
2. 直接使用上游内容作为发布文件（不做过滤、不做测速）
3. spider.jar 下载到仓库，改写成 GitHub Raw 地址 + MD5
4. spider 多源兜底，失败时自动切换备源
5. mirrors.json 提供多 CDN 出口

## 为什么这样设计

### 复杂方案的弊端

- 测速：网络波动导致误判，好源被删
- 关键词过滤：误伤好源，规则难以维护
- 站点复活：增加复杂度，收益有限
- 继承规则：历史改动难以追溯，出问题难排查

### 简单方案的优势

- 稳定：没有复杂的逻辑，不会出错
- 透明：上游变我们就变，可预测
- 易维护：代码少，规则清晰，出问题容易排查
- 快速：不需要测速，不需要多轮过滤

## CDN 镜像

工具会自动生成 mirrors.json，提供多个 CDN 出口：
- GitHub Raw
- jsDelivr
- GhProxy

## 故障处理

- **上游挂了**: GitHub Actions 每 6 小时检查一次，有变化自动更新
- **spider 下载失败**: 自动切换备用源
- **CDN 访问慢**: 用户可以使用 mirrors.json 中的其他 CDN
